# Performance Lab

## Automated Performance Regression Detection Platform

## 1. Project Overview

This project simulates a **DevOps + performance engineering platform** similar to the tooling used by large engineering organizations (e.g., Apple) to monitor application performance during development.

The system automatically:

- Simulates application commits
- Runs automated performance tests
- Stores performance metrics
- Detects regressions statistically
- Visualizes performance trends
- Allows **interactive commit simulation via a web UI**

The platform demonstrates:

- CI/CD automation
- performance benchmarking
- statistical anomaly detection
- observability tooling
- developer productivity infrastructure

## 3. Performance Metrics

Each run generates three core metrics.

| Metric         | Description                         |
|----------------|-------------------------------------|
| CPU Usage      | Simulated CPU utilization (%)       |
| Memory Usage   | Memory consumption (MB)             |
| Execution Time | Time to complete workload (seconds) |

These metrics follow a **probabilistic distribution** to simulate natural variation.

Example:

CPU ~ N(mean, std_dev)

Example configuration:

CPU usage ~ N(30%, 3%)
Memory usage ~ N(400MB, 25MB)
Execution time ~ N(1.5s, 0.2s)

The system allows **controlled drift of these means** to simulate regressions.

Example timeline:

Commit 1 → CPU mean = 30%
Commit 20 → CPU mean = 33%
Commit 40 → CPU mean = 38%

## 5. Continuous Integration Pipeline

CI is implemented using **GitHub Actions**.

### Trigger

Pipeline executes on every commit.

``` yaml
on:
  push:
    branches: [main]
```

### Pipeline Steps

1. Checkout repository
2. Run benchmark simulation
3. Capture metrics
4. Send metrics to database
5. Run regression detection script
6. Output result

Pipeline Flow:

Commit Push
↓
Run Benchmark
↓
Collect Metrics
↓
Store Metrics
↓
Run Regression Detection

InfluxDB is used because it is optimized for **time-series metrics**.

Advantages:

- high write performance
- time-based queries
- natural Grafana integration
- ideal for performance monitoring

The database will run **locally** on the developer machine.

## 7. Regression Detection

Regression detection uses **z-score statistical analysis**.

### Baseline Window

The system calculates a baseline using the **last 20 commits**.

For each metric:

mean = average(last 20 commits)
std_dev = standard deviation(last 20 commits)

### Z-score Calculation

z = (x - μ) / σ

Where:

x = new metric value
μ = baseline mean
σ = baseline standard deviation

### Regression Threshold

| Condition | Result              |
|-----------|---------------------| 
| \|z\| > 2 | Possible regression | 
| \|z\| > 3 | Strong regression   |

Example:

Baseline CPU mean = 30%
Std dev = 2%

New commit CPU = 36%

z = (36 - 30) / 2 = 3

→ Regression detected
